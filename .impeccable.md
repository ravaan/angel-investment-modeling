## Design Context

### Users

Indian angel investors (HNIs, ~5-25Cr net worth) modeling portfolio construction for pre-seed/seed startup investments. They use this tool to make capital allocation decisions — how much to deploy, at what check sizes, across which stages, and what returns to expect. They are sophisticated with numbers but want clarity, not complexity. Context: focused, analytical decision-making sessions — not casual browsing.

### Brand Personality

**Sharp. Precise. Confident.**

The tool should feel like a trusted financial instrument — authoritative without being intimidating, data-dense without being overwhelming. Think Bloomberg terminal clarity meets Zerodha Console simplicity. Every pixel should earn its place.

### Aesthetic Direction

- **Visual tone:** Professional financial tool — clean lines, strong data hierarchy, restrained color usage
- **Reference:** Zerodha Console — Indian financial context, clean portfolio dashboards, zero visual noise, information-first design
- **Anti-references:**
  - NOT a spreadsheet dump — must feel like a crafted product, not Excel-in-a-browser
  - NOT a generic SaaS dashboard — no gratuitous gradients, no card-heavy Tailwind template aesthetics
  - NOT cluttered — despite 67+ inputs and 7 sections, it must breathe and feel calm
- **Theme:** Dark-first (GitHub-inspired). Light and Warm Amber alternatives. Yellow-tinted input cells are a deliberate design choice (Excel familiarity signal)
- **Typography:** System font stack. Tabular numerals throughout. Small but readable (0.78-0.85rem for data). Uppercase muted headers for structure.
- **Color semantics:** Green = base/positive, Blue = bull/accent, Pink = bear/negative. Yellow = editable input. Red = danger/validation error.

### Design Principles

1. **Data density over decoration** — Show maximum information per viewport. Every chart, table, and metric must justify its space. No ornamental elements.
2. **Input clarity** — The 67+ configurable inputs are the product's core. They must be instantly recognizable (yellow highlight), easy to edit, and clearly show their effect on outputs.
3. **Progressive disclosure** — Sections collapse to one-line summaries. The user sees the big picture first, drills into detail on demand. Dashboard stays expanded; others start closed.
4. **Scenario legibility** — Base/Bull/Bear scenarios are the primary lens. Their color coding (green/blue/pink) must be instantly scannable across all tables and charts.
5. **Financial precision** — Indian number formatting (lakhs/crores), proper rupee symbols, tabular numeral alignment, percentage/multiple formatting. No approximation in display — if the model computes 2.147x, show 2.1x, not "about 2x".

### Technical Constraints

- Pure HTML/CSS/JS, no frameworks, no build tools
- Total size under 100KB (currently ~100KB across 6 files)
- SVG charts (no chart libraries)
- Web Audio API for sound (oscillator-based, no audio files)
- Must work on GitHub Pages
- 3 themes via CSS custom properties
- Mobile responsive
- WCAG AA accessibility (ARIA labels, keyboard nav, reduced motion support)
